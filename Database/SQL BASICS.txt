I) NUMERIC TYPES

🔢 1. Exact Numeric Data Types

Used when precision is critical (like for money, counts, etc.).

Data Type	                Storage	                          Range	                                                              Example
int	                        4 bytes	                          -2,147,483,648 to 2,147,483,647        	                        int myAge = 28
smallint	                2 bytes	                          -32,768 to 32,767	                                                smallint empCount = 500
tinyint	                        1 byte 	                           0 to 255	                                                        tinyint retryCount = 3
bigint	                        8 bytes	                          -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807	        bigint population = 7800000000
decimal(p, s) / numeric(p, s)	Variable	                   Fixed precision & scale	                                        decimal(5,2) price = 123.45
bit	                        1 bit (but 8 stored per byte)      0 or 1 (boolean-like)	                                        bit isActive = 1
money	                        8 bytes                           -922,337,203,685,477.5808 to 922,337,203,685,477.5807	                money salary = 56000.75
smallmoney              	4 bytes	                          -214,748.3648 to 214,748.3647                            	        smallmoney bonus = 1500.25

🌐 2. Approximate Numeric Data Types
Used when small rounding errors are acceptable (e.g., scientific calculations).

Data Type	   Storage	     Description	                                Example
float(n)       	   4 or 8 bytes	    Approximate number with floating point	        float temp = 98.6
real	          4 bytes	    Approximate number, less precision than float	real avgMarks = 72.5

ex query :
Create Table Datatypes (
Id int Primary Key,
Name Varchar(100),
Price Decimal(10,2),
Quantity SmallInt,
Discount Real,
isAvailable BIT,
PaidMoney money,
change smallmoney

);


II) STRING TYPES

🔠 1. Character String Data Types (Non-Unicode)

Data Type	    Description	                          Max Length	                    Example
char(n)            Fixed-length non-Unicode string	1 to 8,000 characters	            char(5) always stores 5 characters (pads with spaces)
varchar(n)	   Variable-length non-Unicode string	1 to 8,000 characters or MAX	    varchar(20) stores actual string length
varchar(MAX)	   Variable-length non-Unicode string	Up to 2^31-1 characters (~2GB)	    Used for large text data

📌 Non-Unicode: Stores using 1 byte per character, suitable for English or ASCII data.

🌐 2. Unicode Character String Data Types

Data Type	Description                     	Max Length	                   Example
nchar(n)	Fixed-length Unicode string	        1 to 4,000 characters	           nchar(10) reserves space for 10 Unicode characters
nvarchar(n)	Variable-length Unicode string	        1 to 4,000 characters or MAX	   nvarchar(50) stores Unicode strings
nvarchar(MAX)	Variable-length Unicode string	        Up to 2^30-1 characters (~1GB)	   For storing very large Unicode text

📌 Unicode: Stores using 2 bytes per character, supports global languages like Hindi, Chinese, etc.

ex query : 
  CREATE TABLE Employees (
    EmpID INT,
    FirstName VARCHAR(50),
    LastName CHAR(20),
    Bio NVARCHAR(MAX),
    Nationality NCHAR(10)
);

🔁 Summary Table

Data Type	     Stores	        Precision	Range Start	Size
DATE	             Date only	           -	        0001-01-01	3 B

IV) BINARY DATA TYPES

📦 Binary Data Types in SQL Server

Data Type	Description	                                   Storage
BINARY(n)	Fixed-length binary data (0 to 8000 bytes)	Always exactly n bytes
VARBINARY(n)	Variable-length binary data (0 to 8000 bytes)	Up to n bytes
VARBINARY(MAX)	For very large binary data (up to 2 GB)	        Up to 2,147,483,648 bytes (2 GB)

6. Special Data Types

SQL also supports some specialized data types for advanced use cases:

XML Data Type     :  The XML data type allows for the storage of XML documents and fragments in a SQL Server database.
Spatial Data Type :  A datatype is used for storing planar spatial data, such as points, lines, and polygons, in a database table.


DataType                 Description

XML Datatype             Used to store XML data and manipulate XML structures in the database
Geometry                 stores planar spatial data, such as points, lines, and polygons, in a database table.
